# Clojure Style Guide - Reference Documentation

This directory contains comprehensive reference documentation for the Clojure style skill.

## Structure

**Main Skill File:** `../clojure-style.md`
- Quick reference for code review
- Critical rules highlighted
- Common patterns and idioms
- Consult this during active development

**Reference Files:**

### `stuart-sierra-rules.md`
Complete compilation of Stuart Sierra's "Clojure Don'ts" series:
- Lazy effects (critical)
- Redundant map functions
- Single-branch if statements
- Polymorphism misuse
- Thread-as macro usage
- Anonymous function limitations
- Full explanations with examples and reasoning

### `style-guide.md`
Comprehensive Clojure Style Guide covering:
- Naming conventions (detailed)
- Code organization and structure
- Formatting rules and indentation
- Idiomatic patterns and best practices
- Documentation standards
- Error handling patterns
- Testing conventions
- Performance considerations

## When to Use

**Main skill file (`clojure-style.md`):**

**Reference files:**
